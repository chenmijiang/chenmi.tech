# i18n Implementation Design

## Overview

Add internationalization (English/Chinese) support using Astro's built-in i18n routing with URL prefixes.

## Requirements

1. Support English and Chinese (中文)
2. Default locale (`en`) has **no URL prefix**; Chinese uses `/zh/` prefix
3. Browser language auto-detection via middleware redirect
4. Manual language switcher to toggle between locales
5. All internal links locale-aware (generated via Astro i18n helpers)
6. Blog post content unchanged

## URL Structure

| Locale | Route | Example |
|--------|-------|---------|
| English (default) | No prefix | `/posts`, `/about`, `/search` |
| Chinese | `/zh/` prefix | `/zh/posts`, `/zh/about`, `/zh/search` |

## Architecture

### Astro i18n Config

```javascript
i18n: {
  locales: ["en", "zh"],
  defaultLocale: "en",
  routing: {
    prefixDefaultLocale: false,  // en has no prefix, zh has /zh/ prefix
  },
  fallback: {
    zh: "en",
  },
}
```

### Key URL Generation

```typescript
// Generate locale-aware URLs
import { getRelativeLocaleUrl } from "astro:i18n";

getRelativeLocaleUrl("en", "posts")  // → "/posts"
getRelativeLocaleUrl("zh", "posts")  // → "/zh/posts"
getRelativeLocaleUrl("en", "")        // → "/"
getRelativeLocaleUrl("zh", "")        // → "/zh/"
```

### Middleware (Browser Language Detection)

```javascript
export const onRequest = async (context, next) => {
  const url = new URL(context.request.url);

  // Redirect root to appropriate locale based on Accept-Language
  if (url.pathname === "/" || url.pathname === "") {
    const acceptLanguage = context.request.headers.get("accept-language") || "";
    const locale = acceptLanguage.toLowerCase().startsWith("zh") ? "zh" : "en";
    
    if (locale === "zh") {
      return context.redirect("/zh/", 302);
    }
    // For en, don't redirect - root "/" IS the English page
    return next();
  }

  // Redirect /blog to /posts (existing behavior)
  if (url.pathname.startsWith("/blog/")) {
    return context.redirect("/posts/" + url.pathname.slice(6), 301);
  }
  if (url.pathname === "/blog" || url.pathname === "/blog/") {
    return context.redirect("/posts", 301);
  }

  return next();
};
```

### LanguageSwitcher Component

Uses `getRelativeLocaleUrl()` to generate the alternate locale URL:

```astro
---
import IconGlobe from "@/assets/icons/IconGlobe.svg";
import { getRelativeLocaleUrl, type Locale } from "astro:i18n";

const currentLocale = (Astro.currentLocale || "en") as Locale;
const alternateLocale = currentLocale === "en" ? "zh" : "en";

// Get current path without locale prefix to build alternate URL
const currentPath = Astro.url.pathname.replace(/^\/(en|zh)\//, "/");
const alternatePath = getRelativeLocaleUrl(alternateLocale, currentPath.slice(1));
---

<a href={alternatePath} class="..." title={`Switch to ${alternateLocale === "en" ? "English" : "中文"}`}>
  <IconGlobe class="h-5 w-5" />
</a>
```

### Internal Link Generation Pattern

All internal links must use Astro i18n helpers:

```astro
<!-- ❌ Wrong - hardcoded path -->
<a href="/posts">Posts</a>

<!-- ✅ Correct - locale-aware -->
<a href={getRelativeLocaleUrl(Astro.currentLocale, "posts")}>Posts</a>
```

### getLocalePath Utility

For blog post URLs that need special slug handling (used by getPath internally):

```typescript
// src/i18n/utils.ts
import { getRelativeLocaleUrl, type Locale } from "astro:i18n";

export function getLocalePath(locale: Locale, path: string): string {
  if (locale === "en") {
    return `/${path}`;
  }
  return getRelativeLocaleUrl(locale, path);
}
```

## SEO Considerations

1. **hreflang**: Each page includes alternate hreflang links
2. **canonical**: Each locale has its own canonical URL
3. **sitemap**: Consider generating separate sitemap entries per locale
4. **RSS**: Single feed with hreflang alternate links

## Static Build Behavior

With `output: "static"` and Astro i18n routing:
- `astro build` generates HTML for all locale routes
- Pages under `/posts` are English (default locale)
- Pages under `/zh/posts` are Chinese
- `Astro.currentLocale` returns current locale from URL path
