# i18n Implementation Design

## Overview

Add internationalization (English/Chinese) support to the blog using Astro's native i18n routing with URL prefixes (`/en/` and `/zh/`).

## Requirements

1. Support English and Chinese (中文)
2. Default locale (`en`) has no URL prefix; Chinese uses `/zh/` prefix
3. Browser language auto-detection: redirect root path to appropriate locale
4. Manual language switcher to toggle between locales
5. All UI text translatable, blog post content unchanged
6. SEO-friendly with proper hreflang tags

## Architecture

### URL Structure

```
/                     → 301 redirect to /en/ or /zh/ based on browser language
/en/                  → English (default)
/en/posts             → English posts list
/en/about             → English about page
/zh/                  → Chinese
/zh/posts             → Chinese posts list
/zh/about             → Chinese about page
```

### Key Design Decisions

1. **Default locale (`en`)**: No prefix (`/en/`) - keeps existing URLs working
2. **Chinese (`zh`)**: Uses `/zh/` prefix
3. **Browser detection**: Middleware redirects root to appropriate locale
4. **Language toggle**: Navigates between `/en/` and `/zh/` versions of current page

## Implementation

### astro.config.mjs

```javascript
i18n: {
  locales: ["en", "zh"],
  defaultLocale: "en",
  routing: {
    prefixDefaultLocale: false,
  },
  fallback: {
    zh: "en",
  },
},
```

### Middleware (Browser Language Detection)

```javascript
export const onRequest = async (context, next) => {
  const url = new URL(context.request.url);

  // Redirect root to appropriate locale
  if (url.pathname === "/" || url.pathname === "") {
    const acceptLanguage = context.request.headers.get("accept-language") || "";
    const locale = acceptLanguage.toLowerCase().startsWith("zh") ? "zh" : "en";
    const targetPath = locale === "en" ? "/en/" : "/zh/";
    return context.redirect(targetPath, 302);
  }

  return next();
};
```

### Translation System

Same as previous design - `src/i18n/ui.ts` and `src/i18n/utils.ts` store translation strings.

### LanguageSwitcher Component

```astro
---
import IconGlobe from "@/assets/icons/IconGlobe.svg";
const { lang } = Astro.locals;
const alternateLang = lang === "en" ? "zh" : "en";
const currentPath = Astro.url.pathname;
const alternatePath = currentPath.replace(/^\/(en|zh)\//, `/${alternateLang}/`);
---

<a href={alternatePath} class="..." aria-label="Switch language">
  <IconGlobe class="h-5 w-5" />
  <span class="sr-only">{alternateLang === "en" ? "English" : "中文"}</span>
</a>
```

## SEO Considerations

1. **hreflang**: Add `<link rel="alternate" hreflang="zh" href="/zh/...">` in Layout
2. **canonical**: Each locale has its own canonical
3. **sitemap**: Generate separate sitemap entries per locale
4. **RSS**: Separate feeds per locale or single feed with hreflang

## Static Build Behavior

With `output: "static"` and Astro i18n routing:
- `astro build` generates HTML for all locale routes
- Each page gets its locale from URL path
- `Astro.currentLocale` returns current locale from URL
